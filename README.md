# stack-aws-cloudformation


# Implementando minha Primeira Stack com AWS CloudFormation

## Sobre o desafio

Este repositório foi desenvolvido como parte do laboratório da DIO **Implementando sua Primeira Stack com AWS CloudFormation**.

O objetivo foi aprender como utilizar o AWS CloudFormation para automatizar a criação e o gerenciamento de recursos na AWS através do conceito de **Infrastructure as Code**.

Além da implementação prática, este repositório documenta os principais conceitos estudados, as etapas realizadas e os aprendizados obtidos durante o laboratório.

---

# Objetivos do laboratório

- Compreender o funcionamento do AWS CloudFormation;
- Criar uma Stack utilizando um template;
- Entender como o CloudFormation automatiza a criação de infraestrutura;
- Criar uma Stack contendo regras de Firewall;


---

# Conteúdo estudado

## O que é AWS CloudFormation

Nesta etapa conheci o serviço AWS CloudFormation e seu principal objetivo: permitir a criação, atualização e gerenciamento da infraestrutura da AWS através de arquivos de configuração chamados **Templates**.

Principais conceitos aprendidos:

- Infrastructure as Code (IaC);
- Templates em YAML ou JSON;
- Automação da infraestrutura;
- Versionamento da infraestrutura;
- Reutilização de ambientes;
- Padronização das configurações.

### Benefícios observados

- Evita configurações manuais;
- Reduz erros humanos;
- Facilita recriação de ambientes;
- Permite versionamento da infraestrutura;
- Automatiza implantações.

## Criando Stacks no AWS CloudFormation

Nesta etapa foi realizada a criação de uma Stack utilizando um template.

# Conclusão

Este laboratório permitiu compreender, na prática, como o AWS CloudFormation simplifica a criação e o gerenciamento de infraestrutura por meio da automação. Também foi possível perceber as vantagens da abordagem Infrastructure as Code, que proporciona ambientes mais padronizados, reprodutíveis e fáceis de manter.
